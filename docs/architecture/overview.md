## 技术栈选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| **数据获取** | AkShare | 开源免费、A 股数据全面、更新及时 |
| **数据存储** | SQLite + HDF5 | 轻量级、无需服务、高性能时序数据处理 |
| **数据处理** | NumPy + Pandas | 数据科学生态成熟、向量化计算高效 |
| **可视化** | Plotly | 交互式图表、支持 Streamlit 集成 |
| **前端框架** | Streamlit | Python 原生、快速构建数据应用 |
| **回测引擎** | 自研轻量级 | 完全控制、针对 A 股特性定制 |

## 核心模块关系

### 数据流向
```
AkShare → data_manager → (SQLite/HDF5) → factor_engine → backtest_engine → Streamlit UI
```

### 模块依赖
- **data_manager**：基础模块，无内部依赖
- **factor_engine**：依赖 data_manager
- **sentiment_engine**：依赖 data_manager
- **backtest_engine**：依赖 data_manager、factor_engine、sentiment_engine
- **app.py**：依赖所有业务模块

## 安全设计原则

1. **输入验证**：所有 AkShare API 调用都有超时和重试机制
2. **数据完整性**：SQLite 使用事务保证数据一致性
3. **防未来函数**：所有时间序列操作严格检查数据时效性
4. **错误处理**：模块间有清晰的异常传递和恢复机制
